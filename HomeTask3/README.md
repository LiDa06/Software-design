# Проект 3
## 1. Архитектура системы

Система состоит из трёх микросервисов:

### 1.1. ApiGateway

Главная точка входа для пользователя.
Принимает файл, передаёт его в сервис хранения и запускает анализ.
Возвращает итоговый результат клиенту.

### 1.2. FileStorageService

Отвечает за приём и сохранение присланных файлов.
Сохраняет работу студента и сообщает, где она находится.

### 1.3. FileAnalysisService

Проводит анализ файла, получая его из хранилища.
Возвращает процент совпадений или другие результаты анализа.

Все сервисы работают независимо и общаются между собой через HTTP.

## 2. Пользовательские сценарии
### 2.1. Загрузка файла

Пользователь отправляет работу через ApiGateway.

Gateway передаёт файл в FileStorageService.

После успешного сохранения Gateway вызывает FileAnalysisService.

Анализатор возвращает результат, который Gateway отправляет пользователю.

### 2.2. Получение результата анализа

Пользователь получает:

процент совпадений,

текстовое заключение,

статус обработки.

## 3. Технические сценарии взаимодействия
### 3.1. Последовательность обмена при загрузке файла
Пользователь -> Gateway -> Storage -> Gateway -> Analysis -> Gateway -> Пользователь


Описание шагов:

Gateway получает файл от клиента.

Отправляет его в FileStorageService.

Storage сохраняет файл и сообщает Gateway путь к нему.

Gateway инициирует анализ.

FileAnalysisService запрашивает файл у Storage.

Проводит анализ и возвращает результат Gateway.

Gateway отправляет ответ пользователю.

# 4. Итог

Система реализует простую и понятную микросервисную архитектуру.
Каждый сервис отвечает только за свою задачу:

Gateway — управление запросами,

Storage — хранение,

Analysis — анализ работ.

Такое разделение делает систему гибкой, удобной для расширения и поддерживаемой.
